# `.work/feedback/` — product & operator intake

**Purpose:** Capture user/operator observations, bugs, and feature requests **before** they become SPECs or `NEXT.md` iterations.

| Convention | Rule |
|------------|------|
| Filename | `YYYYMMDD-<slug>.md` |
| Source | Who reported + date + link to artifact (`tmp/`, screenshot, session) |
| Triage | Each item gets `status`: `open` \| `partial` \| `duplicate` \| `done` \| `wontfix` |
| Promotion | High-value items move to `.work/plans/NEXT.md` § Intake queue or a feature SPEC |

**Not for:** session handoff (`HANDOFF.md`), ADRs, or implementation notes — use the paths in `.work/README.md`.
